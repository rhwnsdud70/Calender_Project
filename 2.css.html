2. CSS

  • MY first class

    나의 첫번째 수업에서는 style이라는 태그를 이용하는 법을 배웠다.
    만약 글씨의 색깔을 빨간색으로 하고싶으면 어떻게 해야할까?
    color:red라는 표시를 태그에 넣어주면 될것이다.
    근데 이걸 보면 이게 html을 이용한 것인지 css를 이용한 것인지를 알 수가 없다.
    그렇기에 style 태그를 이용하여 style="color=red"를 이용해 주는 것이다.
    또는 head태그 속에 a 부분의 색깔을 모두 바꾸기 위해 꺽새안에 style을 넣어 style a { color:red; } /style 이런식으로 사용해 줄 수도 있다.
    처음 배웠던 방식이 style의 속성을 이용하는 법이고 뒤의 배운 부분이 style 태그를 이용하는 것이다.
    style의 태그를 이용할때 중요한 것이 있다. 만약 밑줄이 나타나는게 싫어서 text-decoration: none;을 쓴다고 가정하자. 이때 ;이 없다면 문장의 구분이 불가능하기에 문제가 발생한다.

  • MY second class

    a { color:red; } 를 보자 여기서 a는 a를 모두 선택하는 뜻으로 a의 부분을 Selector라고 한다.
    color:red는 말 그대로 Declaration이라고 하고 그 중 color 부분을 Property, red부분을 Value라고 한다.

  • MY third class

    맨 위의 큰 주제로 설정한 Frontend study 라는 글씨를 조금 더 크게, 중앙정렬을 해서 나타내고 싶다고 하자.
    이를 다루기 위해선 내가 원하는 부분을 Selector를 통해 지정해주고 그 안에 Property와 Value를 통해 원하는 형태를 지정해주어야 한다.
    그럼 글씨를 크게 하기 위해선 어떻게 해야할까? 바로 검색해보면 된다.
    글씨의 크기를 설정해 주어야 하니 간단하게 css text size property 이런식으로 말이다.
    우린 이런 property를 외울 필요가 없다. 일련의 과정을 통해 검색하는 법을 알게되었고 이를 통해 1분이면 원하는 방법을 모두 사용할 수 있으니 말이다.
    중앙정렬 또한 동일하다. css text align property value 이런식으로 검색해보면 그만이다.

  • MY fourth class

    현재 내가 페이지에 이용하는 링크인 1.HTML, 2.CSS, 3.JavaScript를 각 상황에 맞는 색으로 표현해보려고 한다.
    예시로 평범하게는 검은색, 한번 들어갔다면 회색, 현재 내가 머물고 있는 페이지는 빨간색으로 말이다.
    먼저 링크로 엮여있는 a에 대해 color:black을 이용해 검은색 글씨로 만들어주었다.
    한번 들어갔을때 회색글씨로 남겨주기 위해, a로 엮인 태그속에 class="saw"로 지정해주고 .이란 단어를 saw앞에 써 .saw를 통해 Selector를 완성시켰다.
    마지막으로 현재 페에지를 빨간색 글씨로 남겨주기 위해 id="active"를 통해 #active를 selector로 지정해주었다.
    이런 style의 일련의 과정에 매우 중요한 것이있다. 바로 우선순위이다.
    ID선택자와 클래스 선택자의 경우 ID선택자가 우선순위를 가진다. 또한 클래스선택자와 태그선택자의 경우 클래스선택자가 우선순위를 가진다.
    만약 같은 우선순위라고 한다면 가장 마지막에 써준 선택자가 제일 높은 우선순위를 가진다.
    그럼 왜일까? id의 경우는 절대 중복되서는 안된다. 즉 유일무이한 값을 가진다는 것이다. 또한 id의 경우 일반적인 태그보다 구체적이다. CSS에선 구체적인 부분이 포괄적인 부분보다 우선순위를 가진다.
    말했듯 구체적이고 포괄적인 관계를 생각하면 헷갈리지 않을 것이다. 잘 모르겠으면 검색해보면 된다. CSS selector priority와 같이 말이다.
